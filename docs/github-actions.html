Despre GitHub Actions

Proiectul include workflows GitHub Actions pentru automatizarea scraping-ului și testării direct din GitHub, fără a necesita configurare locală.

Notă: GitHub Actions este util pentru:
  • Scraping periodic automat (în fiecare luni)
  • Testare automată la fiecare push/PR
  • Deploy continuu la push pe master

Workflows Disponibile

Workflow Descriere Fișier
AROBS Scraper Scrapează job-urile AROBS și le trimite în Solr. Rulează automat în fiecare luni la 6:00 UTC sau manual prin workflow_dispatch. .github/workflows/scrape.yml
Tests Rulează testele Jest la fiecare push sau pull request pe branch-ul master. .github/workflows/test.yml
Deploy Rulează scraper-ul la fiecare push pe branch-ul master. Asigură că datele sunt actualizate după orice modificare. .github/workflows/deploy.yml

1. AROBS Scraper (scrape.yml)

Programare:

  • Cron: 0 6 * * 1 - În fiecare luni la 6:00 UTC (9:00 România)
  • Manual: Prin workflow_dispatch din interfața GitHub

Pași:

1. Checkout repository
2. Setup Node.js 20
3. Instalează dependențe: npm ci
4. Rulează scraper-ul: node index.js

Fișier YAML:

name: AROBS Scraper

on:
  schedule:
    - cron: '0 6 * * 1'  # Every Monday at 6:00 UTC
  workflow_dispatch:

jobs:
  scrape:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: node index.js
        env:
          SOLR_AUTH: ${{ secrets.SOLR_AUTH }}
Important: Workflow-ul folosește ${{ secrets.SOLR_AUTH }} pentru autentificare. Acest secret trebuie configurat în GitHub Settings!

2. Tests (test.yml)

Declanșare:

  • Push pe branch-ul master
  • Pull Request către master

Pași:

1. Checkout repository
2. Setup Node.js 20
3. Instalează dependențe: npm ci
4. Rulează toate testele: npm test

Fișier YAML:

name: Tests

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm test
✓ Nu necesită SOLR_AUTH - Testele unitare nu au nevoie de conexiune la Solr (folosesc mock-uri).

3. Deploy (deploy.yml)

Declanșare:

  • Push pe branch-ul master

Pași:

1. Checkout repository
2. Setup Node.js 20
3. Instalează dependențe: npm ci
4. Rulează scraper-ul: node index.js

Fișier YAML:

name: Deploy

on:
  push:
    branches: [master]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: node index.js
        env:
          SOLR_AUTH: ${{ secrets.SOLR_AUTH }}
Important: Workflow-ul de deploy rulează scraper-ul la fiecare push pe master, asigurând că indexul Solr este mereu sincronizat cu codul.

Cum să Configurezi Secretele

1. Mergi la repository-ul GitHub
2. Mergi la Settings → Secrets and variables → Actions
3. Click pe butonul New repository secret
4. Adaugă secretul:
  • Name: SOLR_AUTH
  • Value: solr:SolrRocks (sau valoarea reală)
5. Click pe Add secret
Notă: Secretele sunt criptate și nu pot fi citite de utilizatori odată salvate. Sunt disponibile doar în timpul execuției workflow-urilor.

Troubleshooting

Problemă Soluție
Workflow-ul eșuează cu "SOLR_AUTH not set" Verifică că secretul SOLR_AUTH este configurat în Settings → Secrets
Job-urile nu sunt în Solr după workflow Verifică log-urile workflow-ului pentru erori la Solr upsert
Workflow-ul de test eșuează Verifică log-urile pentru erori specifice de test
Workflow-ul nu pornește la programare Verifică că branch-ul default este master și că Actions sunt activate
