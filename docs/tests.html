Despre Teste

Proiectul folosește Jest ca framework de testare. Testele sunt împărțite în trei categorii: unitare, integrare și end-to-end.

Notă: Testele sunt configurate cu "type": "module" și folosesc flag-ul --experimental-vm-modules pentru suport ESM în Jest. Configurarea se află în package.json.

Cum să Rulezi Testele

1. Instalează dependențele

Rulează în directorul rădăcină:
npm install

2. Rulează toate testele

npm test

3. Rulează o categorie specifică

npm run test:unit
npm run test:integration
npm run test:e2e

4. Rulează un fișier specific

node --no-deprecation --experimental-vm-modules node_modules/jest/bin/jest.js tests/unit/index.test.js

Configurare Jest

Configurarea se află în package.json:

{
  "jest": {
    "testEnvironment": "node",
    "testMatch": ["**/tests/**/*.test.js"],
    "verbose": true,
    "reporters": ["default", "jest-html-reporter"],
    "testTimeout": 30000
  }
}
Important: Folosim --experimental-vm-modules pentru a permite Jest să ruleze cu ESM (ECMAScript Modules). Toate fișierele sursă folosesc import/export.

Strategia de Mock

Pentru testele unitare, folosim mock-uri pentru API-urile externe:

  • ANAF API - mock la fetch pentru a returna date statice
  • Solr API - mock la fetch pentru a simula răspunsurile Solr
  • Peviitor API - mock la fetch pentru date companie
  • iCIMS HTML - mock cu HTML static pentru parsare cheerio

Testele E2E (scraper.test.js) fac fetch real la iCIMS - nu folosesc mock-uri pentru a verifica conexiunea live.

Adaugă Teste Noi

Pentru a adăuga un test nou:

1. creează un fișier *.test.js în folderul tests/unit/, tests/integration/ sau tests/e2e/
2. Importă funcțiile de testat din modulele sursă
3. Definește testele folosind describe și it
4. Rulează testul pentru a verifica că funcționează

Exemplu:

import { parseJobsPage } from "../../index.js";

describe("parseJobsPage", () => {
  it("parses job title correctly", () => {
    const result = parseJobsPage(sampleHtml);
    expect(result.jobs[0].title).toBe("Software Engineer");
  });
});
