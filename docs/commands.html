Scripturi npm Disponibile

Toate comenzile se rulează din directorul rădăcină al proiectului.

Comandă Descriere
npm test Rulează toate testele Jest (unitare, integrare, E2E)
npm run scrape Rulează scraper-ul complet (validare ANAF → scraping iCIMS → Solr)
npm run test:unit Rulează doar testele unitare
npm run test:e2e Rulează doar testele end-to-end (necesită conexiune la iCIMS)
npm run test:integration Rulează doar testele de integrare
npm run validate Rulează validatorul de URL-uri job
npm run demoanaf Testează API-ul ANAF cu CIF 11291045

npm run scrape

Rulează scraper-ul complet pentru AROBS Transilvania Software.

Utilizare:

npm run scrape

Echivalent cu: node --no-deprecation index.js

Pași:

1. Validează compania prin ANAF API (CIF 11291045)
2. Verifică job-urile existente în Solr
3. Fetch pagină iCIMS careers și parsează cu cheerio
4. Iterează prin toate paginile de job-uri
5. Extrage tag-uri (seniority + category + skills)
6. Mapare locații (RO-Cluj Napoca → Cluj-Napoca)
7. Salvează datele în tmp/jobs.json
8. Trimite job-urile către Solr (upsert)
9. Actualizează compania în Solr (lastScraped)
Notă: Necesită variabila de mediu SOLR_AUTH configurată pentru a trimite date la Solr.

npm test

Rulează toate testele Jest din proiect.

Utilizare:

npm test

Echivalent cu: node --no-deprecation --experimental-vm-modules node_modules/jest/bin/jest.js

Teste incluse:

📦 Unitare: index.test.js, company.test.js, solr.test.js, demoanaf.test.js, topics.test.js, public.test.js
🔗 Integrare: workflow.test.js (tag-uri, locații, workmode)
🌐 E2E: scraper.test.js (fetch real iCIMS)
Notă: Rezultatul testelor este generat și în HTML prin jest-html-reporter.

node index.js [opțiuni]

Rulează scraper-ul direct cu Node.js.

Opțiuni:

Opțiune Descriere
--test Scrapează doar prima pagină (mod test rapid)

Exemple:

# Rulare completă
node --no-deprecation index.js

# Rulare doar prima pagină (test)
node --no-deprecation index.js --test

node validate-jobs.js

Validează URL-urile job-urilor din Solr sau din fișiere.

Utilizare:

# Validează job-urile AROBS din Solr
node validate-jobs.js 11291045

# Validează și șterge job-urile expirate
node validate-jobs.js 11291045 --delete

# Verifică un URL specific
node validate-jobs.js --url "https://careers-arobs.icims.com/jobs/3444/job"

# Verifică URL-uri dintr-un fișier JSON
node validate-jobs.js --file tmp/jobs.json

node demoanaf.js

Interfață CLI pentru API-ul ANAF.

Utilizare:

# Obține detalii companie după CIF
node demoanaf.js 11291045

# Caută companii după brand
node demoanaf.js search "AROBS"

node solr.js

Operații directe cu Solr pentru mentenanță.

Utilizare:

# Verifică job-urile dintr-un CIF
node solr.js 11291045

# Extrage job-urile în fișier backup
node solr.js extract 11291045

# Interoghează companii
node solr.js company "company:AROBS*"

Configurare SOLR_AUTH

Autentificarea la Solr se face prin variabila de mediu SOLR_AUTH.

Fișier .env.local

# Conținut fișier .env.local
SOLR_AUTH=solr:SolrRocks

Export direct în shell

export SOLR_AUTH="solr:SolrRocks"
node index.js

În GitHub Actions

env:
  SOLR_AUTH: ${{ secrets.SOLR_AUTH }}
Important: SOLR_AUTH este în format username:password. Este codat Base64 și trimis ca Basic Auth la Solr.

Cum Funcționează

Toate modulele sunt scrise în ESM (ECMAScript Modules) cu import/export modern. Proiectul are "type": "module" în package.json.

Lanțul de execuție:

CLI → index.js main()
    → company.js validateAndGetCompany()
        → src/anaf.js getCompanyFromANAF()
        → solr.js querySOLR()
    → scrapeAllListings()
        → fetchPage()
        → parseJobsPage()  (cheerio)
    → mapToJobModel()
    → solr.js upsertJobs()
    → solr.js upsertCompany()
