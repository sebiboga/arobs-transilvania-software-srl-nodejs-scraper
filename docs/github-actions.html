Despre GitHub Actions

Proiectul include workflows GitHub Actions care permit scraping-ul direct din GitHub, fără a necesita configurare locală. Aceste workflows rulează pe servere Ubuntu și folosesc Chrome pentru automatizare.

Notă: GitHub Actions este util pentru:
  • Scraping ocazional al unei companii
  • Automatizarea fără configurare locală
  • Scraping de pe orice calculator cu acces la GitHub

Workflows Disponibile

Workflow Descriere Fișier
opencode_scraper_to_solr Scraper automat pentru o companie specifică - scrapează job-uri și le trimite în Solr .github/workflows/opencode_scraper_to_solr.yml
opencode_cli Rulare OpenCode CLI direct în GitHub Actions .github/workflows/opencode_cli.yml
pages Deploy documentației HTML pe GitHub Pages .github/workflows/pages.yml

Cum să Scraperzi o Companie prin GitHub Actions

Pași:

1. Asigură-te că ai Secrets configurate

În repository-ul GitHub, trebuie să adaugi secretele pentru Solr:

  • Mergi la Settings → Secrets and variables → Actions
  • Adaugă un secret nou: SOLR_USER (username-ul tău Solr)
  • Adaugă un secret nou: SOLR_PASSWD (parola ta Solr)
2. Rulează Workflow-ul
  • Mergi la tab-ul Actions în repository
  • Selectează OpenCode Scraper to Solr
  • Click pe butonul Run workflow
  • Introdu numele companiei (ex: AROBS)
  • Click pe Run workflow
3. Verifică rezultatele

După ce workflow-ul rulează:

  • Click pe workflow run pentru a vedea progress-ul
  • Verifică log-urile pentru numărul de job-uri scrape-ate
  • Verifică în Solr că job-urile au fost adăugate
Ce face workflow-ul automat:
  • Instalează Chrome și OpenCode
  • Rulează comanda /scrape pentru compania specificată
  • Pune job-urile în Solr
  • Dacă se creează un nou fișier scraper (ex: zira-marketing.md), îl commită automat în repo
  • Actualizează câmpurile lastScraped și scraperFile în Solr

Exemplu de Rulare

Workflow Dispatch cu input:

Company name: AROBS

Rezultat așteptat:

✓ Scraped jobs for AROBS
✓ Pushed jobs to Solr
✓ Updated company in Solr (lastScraped: 2026-06-06)

Cum Funcționează

Fluxul de Lucru:

1. Workflow trigger (manual)
        ↓
2. Setup: Chrome + Node.js + OpenCode
        ↓
3. Run /scrape COMMAND (passed as input)
        ↓
4. AI navighează la careers page companiei
        ↓
5. Extrage job-urile (Romania only)
        ↓
6. Push company → Solr company core
        ↓
7. Push jobs → Solr job core
        ↓
8. Check pentru fișiere noi scraper
        ↓
9. Commit + Push fișier scraper nou (dacă există)
        ↓
10. Update lastScraped + scraperFile în Solr
Important: Workflow-ul folosește ${{ secrets.SOLR_USER }} și ${{ secrets.SOLR_PASSWD }} pentru autentificare. Acestea trebuie configurate în GitHub Secrets!

Troubleshooting

Probleme Comune:

Problem Soluție
Workflow-ul eșuează cu "Chrome not found" Verifică că Chrome este instalat corect pe runner-ul GitHub
Nu se pot trimite date la Solr Verifică că secretele SOLR_USER și SOLR_PASSWD sunt configurate corect
0 jobs scraped Verifică că URL-ul careers page este corect în Solr
Fișierul scraper nu se creează Verifică că workflow-ul are permisiuni de write la repo

Avantaje vs. Metoda Locală

GitHub Actions Scraping Local
Nu necesită instalare Chrome/OpenCode Necessită Chrome + PowerShell + Docker
Rulează de oriunde Rulează doar de pe calculatorul local
Automatizare completă (inclusiv commit) Trebuie să faci commit manual
Limitări: minute GitHub Actions Nelimitat
